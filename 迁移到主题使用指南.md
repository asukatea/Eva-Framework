# Eva Framework · 迁移到主题使用指南

> 目标读者：把 Eva（CSF 替代框架）从「独立插件」搬进 **Lentasy 主题** 内嵌使用、并逐步替换现有 CSF 配置的人。
> 版本基准：Eva `0.1.0`。本文所有结论均来自对源码的核对（`eva-framework.php` / `includes/` 下各 class，按 core/admin/containers/demo 分目录）。

---

## 0. 一句话结论（TL;DR）

- **demo 可删**：`includes/demo-options.php` 及 `eva-framework.php` 末尾对它的 `require` 属于演示，删掉不影响框架本体。
- **能搬进主题**：Eva 幂等加载（`EVA_FW_LOADED`），可像 CSF 那样被主题 `require` 内嵌。
- **迁移本质**：① 主题里 require Eva 本体 → ② `\CSF::createXxx` 改成 `\Eva::createXxx` → ③ **前台取值层基本不用动**（同样存 `wp_options` / `post_meta` / `term_meta`，键名沿用容器 id）。
- **但别急着全量迁**：当前字段类型 / 后端 sanitize 不全（详见 §6），需**边补边迁**。

## 1. 现状与前置认知

| 项 | 说明 |
| --- | --- |
| 位置 | `wp-content/plugins/Eva framework/` |
| 入口 | `eva-framework.php` |
| 命名空间 | 渲染/容器类在 `Eva\Framework\*`；对外注册门面是**全局类** `\Eva` |
| 幂等加载 | 定义 `EVA_FW_LOADED` 常量，重复 `require` 自动 `return`，可安全内嵌主题 |
| 数据存储 | 设置页 → `wp_options[$id]`；metabox → `post_meta`；分类法 → `term_meta`；导航菜单 → 菜单项 `post_meta`（与 CSF 一致） |
| 资源 | Vue3 + Remixicon 走 jsDelivr CDN；`assets/eva-app.js` 为前端外壳；`Fields/*.js` 一字段一组件 |

## 2. 把 Eva 搬进主题（两种方式）

### 方式 A · 保留为插件（最省事，建议迁移期先用）

不搬代码，启用插件后 `\Eva` 类已全局可用，主题任意位置直接调用 `\Eva::createOptions()` 即可。
- 优点：零改造，CSF 与 Eva 可并存，逐模块切换。
- 缺点：依赖插件「已启用」状态，不适合对外发布的主题。

### 方式 B · 内嵌进主题（与 CSF 同款，发布主题用）

1. 复制框架本体到主题，例如：
   `wp-content/themes/Lentasy/Library/eva-framework/`（保留 `includes/` 与 `assets/`）。

2. 在 `loader.php` 的 `load_library()` 中，仿照现有 CSF 那一行加载（迁移期两者共存）：

```php
//加载 Codestar 框架（迁移完成后可移除）
require_once LF_THEME_DIR . LF_DS . 'Library' . LF_DS . 'codestar-framework' . LF_DS . 'codestar-framework.php';

//加载 Eva 框架
require_once LF_THEME_DIR . LF_DS . 'Library' . LF_DS . 'eva-framework' . LF_DS . 'eva-framework.php';
```

3. 内嵌时注意 `eva-framework.php` 里的三处：
   - `register_activation_hook(__FILE__, ...)`：主题内嵌没有插件激活钩子，**不会触发**，无害；伪静态 flush 由 `Standalone` 的「规则签名」机制兜底。
   - `EVA_FW_DEV`：**生产务必关闭**（见 §8）。
   - demo 的 `require`：内嵌前请按 §7 删除。

> 提示：`EVA_FW_DIR` / `EVA_FW_URL` 用 `plugin_dir_path/url(__FILE__)` 推导，放在主题子目录里同样成立，无需改路径常量。

## 3. 注册 API 对照表（CSF → Eva）

API **同名同签**，把前缀 `\CSF::` 换成 `\Eva::` 即可：

| 容器 | CSF | Eva | 存储位置 |
| --- | --- | --- | --- |
| 设置页 | `CSF::createOptions` | `Eva::createOptions` | `wp_options[$id]` |
| 分组 | `CSF::createSection` | `Eva::createSection` | （归属所在容器） |
| 文章 metabox | `CSF::createMetabox` | `Eva::createMetabox` | `post_meta` |
| 分类法 | `CSF::createTaxonomyOptions` | `Eva::createTaxonomyOptions` | `term_meta` |
| 导航菜单 | `CSF::createNavMenuOptions` | `Eva::createNavMenuOptions` | 菜单项 `post_meta` |
| 用户资料 | `CSF::createProfileOptions` | `Eva::createProfileOptions` | `user_meta` |
| 评论 | `CSF::createCommentMetabox` | `Eva::createCommentMetabox` | `comment_meta` |
| 定制器 | `CSF::createCustomizeOptions` | `Eva::createCustomizeOptions` | `wp_options[$id]` |
| 短代码 | `CSF::createShortcoder` | `Eva::createShortcoder` | — |
| 小工具 | `CSF::createWidget` | `Eva::createWidget` | widget 实例 |

### `createOptions` 参数差异（重点）

CSF 专有键 → 迁移时**删除或改写**；Eva 新增了独立页能力：

```php
// CSF 写法（主题现状，Modules/Settings/Main.php）
\CSF::createOptions('Leantasy', array(
    'framework_title' => '...',          // ✗ Eva 无，删
    'menu_title'      => 'LF主题设置',     // ✓ 保留
    'menu_slug'       => 'Leantasy',     // ✓ 保留
    'menu_icon'       => 'https://...',  // ✓ 保留
    'class'           => '...',          // ✗ 删
    'menu_type'       => 'menu',         // ✗ 删（用 location 代替）
    'show_bar_menu'   => false,          // ✗ 删
    'theme'           => 'light',        // ✗ 删
));

// Eva 写法
\Eva::createOptions('Leantasy', array(
    'menu_title'  => 'LF主题设置',
    'menu_slug'   => 'Leantasy',
    'menu_icon'   => 'dashicons-admin-customizer', // 或图片 URL
    'location'    => 'admin_bar',  // admin_bar=顶部工具栏入口；left=左侧菜单
    'standalone'  => true,         // true=独立页(脱离 /wp-admin/)；false=后台全屏页
    'brand'       => 'Leantasy',   // 侧栏品牌名
    'subtitle'    => '...',
    'menu'        => array(),       // 左侧主菜单项；也可用 \Eva::addMenuItem() 追加
));
```

## 4. 接入主题的三段式加载

主题 `loader.php → load_modules()` 现有三阶段，迁移时**流程不变**，只把内部 `\CSF::` 改 `\Eva::`：

1. **阶段一** `init_framework()`：`\Eva::createOptions(...)` 建容器。
2. **阶段二** 加载扩展模块：扩展通过 `do_action('Extend_Settings')` 等钩子注册分组。
3. **阶段三** `init_sections()`：各模块 `\Eva::createSection(...)` 注册字段；`\Eva::createTaxonomyOptions(...)` 建分类法容器。

> 容器 id 不要与分类法 id 同名（主题已用 `Leantasy` 与 `Leantasy_Taxonomy` 区分），Eva 同样按 id 索引，沿用即可。

## 5. 字段写法与取值

### 字段数组

```php
\Eva::createSection('Leantasy', array(
    'id'     => 'general',
    'title'  => '常规设置',
    'icon'   => 'ri-equalizer-line',
    'fields' => array(
        array('id' => 'site_slogan', 'type' => 'text',     'title' => '站点标语', 'desc' => '...', 'default' => ''),
        array('id' => 'enable_x',    'type' => 'switcher',  'title' => '开关',     'default' => false),
        array('id' => 'mode',        'type' => 'select',    'title' => '模式',     'options' => array('a' => 'A', 'b' => 'B')),
    ),
));
```

- **键名差异**：CSF 的 `subtitle`（标题下小字）≈ Eva 的 `desc`；`width` 支持 `1/2`、`1/3`、`full`。

### 取值（迁移友好点）

Eva 与 CSF **同样把设置页存进 `wp_options`，键 = 容器 id，值 = `[field_id => value]`**。
因此主题现有读取层 **无需改动**：

```php
// 主题现成 helper（Modules/Settings/Main.php），内部就是 get_option('Leantasy')[$key]
\LF\Modules\Settings\Main::get_option('site_slogan');
\LF\Modules\Settings\Main::get_term($term_id, 'key');   // term_meta
\LF\Modules\Settings\Main::get_met_box($post_id, 'key', 'slug'); // post_meta
\LF\Modules\Settings\Main::get_nav($item_id, 'key');    // 菜单项 post_meta
```

框架自带读取（等价）：

```php
\Eva::get_values('Leantasy');     // 取整组数组
get_option('Leantasy');           // 原始
```

> 结论：**前台 / 模板层基本零改动**，迁移主要发生在「注册端」。

## 6. 当前差距与迁移顺序（务必先读）

> 这是 0.1.0 的真实短板，**不读会丢数据**。

1. **后端 sanitize 不全**：`Data::sanitize_field()` 目前只认 `text / textarea / select / switcher`，其余一律走 `default → sanitize_text_field((string)$val)`。
   → `group / repeater / fieldset / tabbed / accordion / gallery / checkbox` 等**值为数组**的字段，存盘会被强转成字符串、**数据损坏**。
   → **迁任何含数组型字段的模块前，先扩 `Data::sanitize_field()`**（至少：数组值递归清洗 / 按类型分支）。
2. **前端组件不全**：`Fields/` 仅 `text / textarea / select / switcher / html / backup` 6 种；注册其它 `type` 前台**渲染不出**。
3. **主题实际用到、Eva 暂缺的约 20 种**：
   - 简单输入：`number / radio / checkbox / button_set / spinner / subheading / content`
   - 媒体颜色：`upload / gallery / image_select / icon / color / color_group / palette`
   - 嵌套容器（最难）：`fieldset / group / repeater / tabbed / accordion / sorter`

### 推荐迁移顺序

1. **试点模块**：挑字段少、类型简单的真实模块（如「常规设置 / Routine」），补齐它需要的字段（组件 + sanitize 分支），整模块迁过去，端到端验证：**注册 → 渲染 → 提交 → 存库 → 回读**。
2. 试点跑通后，按「字段类型出现频率」补：先广覆盖的简单输入类，每补一类就迁用到它的一批分区。
3. 媒体 / 颜色类居中。
4. **嵌套容器放最后**——最复杂，且要等简单字段契约定死后做，风险最低。

## 7. demo 删除清单（迁移时务必删）

> **`includes/demo-options.php` 只是一个演示**：它注册了名为 `eva_demo` 的示例设置页（菜单树、几个示例字段、帮助/版本/备份占位），用来展示 Eva 怎么用，**不是主题正式配置**。
> **迁移到主题时应当删除它**，改用主题自己的容器 id（如 `Leantasy`）注册真正的设置；留着它只会多出一个无用的「Eva Framework」演示菜单。

删除步骤：

1. 删文件：`includes/demo-options.php`
2. 删 `eva-framework.php` 末尾这段 `require`（这是唯一加载入口，删了 demo 就彻底不加载）：

```php
// 内置演示设置页（可删除此 require 与 includes/demo-options.php）
require_once EVA_FW_DIR . 'includes/demo-options.php';
```

3. 确认没有别处引用演示容器 id `eva_demo`（迁移到主题后应改用主题真实容器 id，如 `Leantasy`）。

> 删除 demo **不影响框架本体**：`\Eva` 门面（`includes/class-eva.php`）与各容器渲染/保存逻辑（`includes/core`、`includes/admin`、`includes/containers`）与 demo 无依赖关系。

## 8. 上线检查清单

- [ ] `wp-config.php` 设 `define('EVA_FW_DEV', false);`（关闭 live-reload 与 mtime 缓存击穿，资源版本回落到 `EVA_FW_VERSION`）。
- [ ] demo 已按 §7 删除。
- [ ] 已迁模块的字段类型，前端组件与后端 sanitize **都已具备**（否则该字段不可用 / 会丢值）。
- [ ] 评估 CDN 依赖（Vue3 / Remixicon 走 jsDelivr）：内网或强稳定性场景建议本地化到主题 `Assets/`。
- [ ] 迁移期 CSF 与 Eva 共存时，确认同一设置不会被两套框架重复注册到同一菜单 / 同一 option 键。

## 9. 最小可用示例（主题内）

```php
add_action('init', function () {
    if (! class_exists('\Eva')) { return; } // 容错：Eva 未加载则跳过

    \Eva::createOptions('Leantasy', array(
        'menu_title' => 'LF主题设置',
        'menu_slug'  => 'Leantasy',
        'location'   => 'admin_bar',
        'standalone' => true,
    ));

    \Eva::createSection('Leantasy', array(
        'id'     => 'general',
        'title'  => '常规设置',
        'fields' => array(
            array('id' => 'site_slogan', 'type' => 'text', 'title' => '站点标语', 'default' => ''),
        ),
    ));
});

// 模板里取值（与 CSF 时代写法一致）
echo esc_html(\LF\Modules\Settings\Main::get_option('site_slogan'));
```
